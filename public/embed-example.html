Typical Brand Setup Workflow (Before Embedding)

Before you can embed the chat widget, your brand needs to be set up on our platform. Here's a general overview of the steps a brand representative would typically follow in their dedicated admin dashboard:

  1. Account Creation & Sign-up: Register your company on our platform to create a new tenant account. This will give you access to your brand-specific admin dashboard.
  2. Initial Configuration:
    • Upload your company logo.
    • Enter your brand name and a brief company description.
    • Specify your primary website URL (e.g., https://yourbrand.com). The AI will use this as a key context for information.
    • (If applicable) Select a subscription plan.
  3. AI Agent Customization:
    • Define your chat agent(s), including their name (e.g., "BrandX Sales Pro") and a detailed role description (this heavily influences AI behavior).
    • (Advanced/Future) Potentially input specific prompts, key selling points, or company-specific jargon.
  4. Obtain Tenant ID: Your unique tenantId will be available in your admin dashboard. This ID is crucial for linking the embeddable widget to your specific brand's configuration.

Once these steps are completed, your chat widget is configured, and you can proceed to embed it on your website using the snippet below.

To embed the chat widget on your website, copy and paste the HTML snippet below into your webpage's HTML where you want the widget to appear.

Recommended: Floating Widget Snippet

Add this one-liner near the end of your siteâ€™s <body>. Replace YOUR_CHAT_APP_BASE_URL and YOUR_TENANT_ID.


<script src="YOUR_CHAT_APP_BASE_URL/widget.js?tenantId=YOUR_TENANT_ID" defer></script>

Customization Notes:

  • src Attribute:
    • Replace YOUR_CHAT_APP_BASE_URL with the URL where the main chat application is hosted (e.g., https://your-chat-platform.com).
    • Replace YOUR_TENANT_ID with the unique ID assigned to your brand/tenant (e.g., tenant_acme_corp). You get this from your admin dashboard after completing the setup workflow described above.
  • width and height: Adjust these values (e.g., width="380" height="550") to best fit the layout of your website.
  • title Attribute: It's highly recommended to provide a descriptive title for accessibility purposes (e.g., "BrandX Customer Support Chat").
  • style Attribute: Basic styling is provided for a modern look. You can customize this further. For significant visual consistency, theme customization might be offered within your tenant admin dashboard.
  • allow="microphone": This is crucial if your chat widget uses voice input features. Ensure your chat application also correctly requests microphone permissions from the user.

Important Considerations for Embedding:

  • Hosting: The main chat application (at YOUR_CHAT_APP_BASE_URL) must be hosted and publicly accessible for the iframe to load it.
  • Tenant Configuration Complete: Ensure your tenant account (identified by YOUR_TENANT_ID) is fully configured as per the "Typical Brand Setup Workflow" section above. This includes branding, agent definitions, and AI context (like your website URL).
  • Cross-Origin Resource Sharing (CORS) & Content Security Policy (CSP):
    • The server hosting the chat application (YOUR_CHAT_APP_BASE_URL) may need to be configured to allow embedding.
    • Specifically, it might need to send a Content-Security-Policy HTTP header with the frame-ancestors directive specifying the domains that are allowed to embed it (e.g., frame-ancestors https://yourbrand.com;).
    • Consult your hosting provider or server administrator if you encounter issues with the iframe not loading due to security policies.

Example with Placeholders Filled:

<script

  defer
></script>
